# TradeSkills 3 分钟 Demo 讲稿

本文是一份适合现场演示或录屏演示使用的 3 分钟中文讲稿，默认观众是“懂产品但不一定懂代码”的技术管理者、业务同学或潜在合作方。

## 1. Demo 目标

在 3 分钟内，让观众理解 4 件事：

1. TradeSkills 不是普通回测页面，而是“自然语言 Skill -> Agent 运行”的闭环系统
2. 同一份 Skill 可以同时用于回测和实时信号
3. Agent 不是凭空拍脑袋，而是在工具约束下做结构化决策
4. 系统能保存过程、结果和组合变化，便于复盘和演示

## 2. 建议的演示准备

建议提前准备好以下内容：

- Web、API、Runner 三个服务都已启动
- 本地历史数据已经导入
- 使用示例 Skill：`packages/shared-skill-examples/demo-short-sell-skill.md`
- 页面上至少已经能看到：
  - 市场数据状态正常
  - 已上传 Skill 或可现场上传
  - 可以发起回测
  - 可以看到 recent signals / traces

## 3. 推荐演示路线

最稳的一条路线是：

1. 先展示 Skill 长什么样
2. 再展示系统把 Skill 变成了什么
3. 再发起一个回测
4. 最后展示 live signal 或最近信号记录

这样逻辑最顺，也最容易在 3 分钟内把价值讲清楚。

## 4. 3 分钟讲稿

## 0:00 - 0:30 开场

建议讲稿：

“我先用 3 分钟介绍一下 TradeSkills。它的核心不是让用户先写量化代码，而是先写一份自然语言的交易 Skill。比如这份策略写的是：每 15 分钟扫描一次 OKX 永续合约，寻找短期涨得过热、适合做空的币种，结合 EMA、RSI、ATR、持仓量变化来判断，并且限制最大仓位和止损。”

此时建议操作：

- 打开 Skill 列表或直接展示示例 Skill 内容
- 快速滚到节奏、市场扫描、AI reasoning、risk control 这几个部分

## 0:30 - 1:10 说明“为什么这不是普通配置页”

建议讲稿：

“这里的重点不是填写一堆固定参数，而是用 Markdown 把策略逻辑描述出来。系统会自动抽取它的执行节奏、风险约束、需要哪些工具，以及它可以运行在回测还是实时模式。也就是说，这份文本不是文档，而是可执行策略的入口。”

此时建议操作：

- 切到 Skill 详情页
- 指给观众看 envelope、工具约束、trigger cadence、risk contract

如果页面上有工具约束，建议补一句：

“比如这里可以看到，它会需要市场扫描、K 线读取、状态读写这些工具能力。”

## 1:10 - 2:00 演示回测

建议讲稿：

“接下来我直接拿这份 Skill 发起一次回测。系统会按这个 Skill 的执行节奏，把历史时间逐步往前推进。每到一个触发点，Agent 只能看到当时以前的数据，然后调用工具做一次判断，最后产出结构化决策，例如开仓、减仓、平仓或者跳过。”

此时建议操作：

- 发起一个短时间窗回测
- 等待结果出现
- 打开回测详情、summary 或 traces

回测完成后继续讲：

“这里我们看到的不只是一个收益结果，还能看到每一步 Agent 是什么时候触发、做了什么决策、为什么这么做，以及决策前后的组合变化。这一点很重要，因为它让 Agent 的行为是可复盘、可解释的。”

如果有 trace 页面，建议强调：

- reasoning summary
- tool calls
- portfolio before / after
- fills

## 2:00 - 2:40 演示 live signal

建议讲稿：

“同一份 Skill 不只能做历史验证，也可以切到实时模式。实时模式下，系统会按同样的节奏周期性触发这份 Skill，持续输出结构化信号。它和回测用的是同一份策略文本、同一套工具语义，所以研究和运行之间的偏差会更小。”

此时建议操作：

- 切到 live task 或 recent signals 区域
- 展示最近一条 signal

补一句边界说明：

“当前这个 demo 更接近定时运行的实时信号系统，还不是完整实盘交易系统，但已经把策略表达、Agent 决策、组合更新和结果留痕这条链路跑通了。”

## 2:40 - 3:00 收尾总结

建议讲稿：

“所以如果用一句话总结，TradeSkills 做的是把自然语言策略变成可运行的 Agent Skill：用户先写策略文本，系统把它结构化，再让 Agent 在工具约束下执行，同一份 Skill 既能回测，也能周期运行，还能把整个决策过程保存下来，方便调试、复盘和对外展示。” 

最后可以顺势接一句：

“如果你感兴趣，我接下来可以继续展开两部分：一是技术架构怎么做的，二是这套产品未来怎么走向更完整的策略平台。”

## 5. 演示时最值得强调的 5 个点

- 不是先写代码，而是先写 Skill
- 不是黑盒聊天，而是带工具的结构化决策
- 不是只看收益，而是能看全过程
- 同一份 Skill 同时服务回测和实时模式
- 当前已经闭环，但仍保留向更强执行平台演进的空间

## 6. 如果现场时间更短，可以用 60 秒极简版

“TradeSkills 是一个把自然语言交易策略变成可运行 Agent 的平台。用户上传一份 Markdown Skill，系统会自动抽取执行节奏、风险约束和工具需求，然后让 Agent 在统一工具框架下运行。它既可以做历史回测，也可以按固定节奏输出实时信号，而且会记录每一步为什么这么决策、调用了什么工具、组合怎么变化，所以它既能做策略验证，也很适合做可解释的产品演示。”

## 7. 如果现场有人追问，可以这样接

### 问：这是不是已经在做实盘自动交易？

建议回答：

“当前版本更聚焦在策略表达、回测验证和实时信号闭环，还不是完整实盘自动交易系统。现在最重要的是先把 Agent 的决策过程和运行链路跑通并可解释化。”

### 问：这和普通量化回测平台有什么不同？

建议回答：

“普通平台通常默认你已经有了程序化策略；TradeSkills 更强调前置阶段，也就是怎么把自然语言策略快速变成可运行 Agent，并且保持回测和实时模式的一致性。”

### 问：这是不是多 Agent 系统？

建议回答：

“当前不是多 Agent 编排，而是一个统一的交易执行 Agent Runtime。下一步才比较自然地扩展到研究 Agent、交易 Agent、风控 Agent 等多角色模式。”
